1. 프로젝트 개요 — 원칙 · 리스크

설계 3원칙, 타깃 유저, 알려진 리스크를 정리함.

설계 3원칙

  • ① 맵 선택이 중심 — 다음 목표를 명확히 고르는 UX를 우선한다
  • ② 변신 = 플레이 스타일 — 단순 스킨이 아닌 전략 선택지
  • ③ MVP 우선 — 출시 시 뺀 것이 넣은 것보다 많아야 함

타깃 유저

  • 방치형 RPG 선호 + 단순 성장만으로는 심심함 느끼는 유저
  • 혼자 성장하면서 협동 목표도 원하는 유저
  • 가챠 / 합성 수집 재미를 즐기는 유저

알려진 리스크

  • 콘텐츠 축이 많아지면 4인 팀 기준 완성도가 먼저 무너짐
  • 변신 시스템이 과도하게 복잡해지면 BM과 UI가 동시에 무거워짐
  • 맵 선택 UX가 약하면 일반 방치형과의 차별점이 흐려짐
  • 테마 확정 전 특정 세계관 표현을 과도하게 문서화하면 수정 비용이 커짐

2. 출시 스코프 — 포함 항목

출시에 들어가는 콘텐츠, 성장 시스템, 핵심 수치를 개발 기준으로 정리함.

출시 포함 — 콘텐츠

  • 기본 싱글 사냥터 (스테이지 1~10000)
  • 맵 선택 화면 (기본/월드 보스/협동/던전 분기)
  • 월드 보스 1종 (시간제 입장)
  • 협동 사냥터 1종 (1~4인 매칭, Photon Fusion) + 싱글 채널
  • 던전 카테고리 5종 (구슬/경험치/소환/아티팩트/유물) — 노출만, 상세 규칙 별도

출시 포함 — 성장 시스템

  • 장비 8부위 (구슬 뽑기, 등급 8단계, 주옵 2 / 부옵 2)
  • 변신 뽑기 (일반~영웅 소환, 전설~신화 합성 전용)
  • 스킬 슬롯 5개 (액티브 전용, 뽑기 레벨 0~15)
  • 펫 시스템 (메인 1 + 서브 4, 6등급, 뽑기 레벨 0~15)
  • 특수 장비 2종 (날개 / 인장, 협동 전용 성장축)

월드 보스 핵심 수치

  • 입장 가능 시간: 하루 4회 — 02:00 / 08:00 / 14:00 / 20:00
  • 일일 도전 제한: 2회
  • 보스 종류: 1종 (출시 기준)
  • 보상 구조: 고정 보상 + 낮은 확률 추가 보상 + 딜량 보상
  • 패배 처리: 일반 몬스터 10킬부터 재시작 (스테이지 유지)
  • 추천 전투력: 표시 안 함 (출시 기준)

협동 핵심 수치

  • 주간 이용 제한: 7시간 (실제 입장 시간 누적 차감)
  • 오픈 시간: 하루 2회 — 13:00~15:00 / 20:00~22:00 (KST)
  • 난이도: 5단계 유저 선택
  • 최대 인원: 4명 (싱글 채널 별도)
  • 기여도 판정: 협동 전용, 몬스터별 독립, 최대 체력 기준 40% 이상 누적 피해 시 해당 몬스터 보상 획득
  • 중간 입장 / 퇴장: 자유, 불이익 없음

오프라인 보상 규칙

  • 광고 시청: 1.5배 획득 선택 가능

7. 전투 시스템 — 전투 컨셉

전투 철학

플레이어는 전투를 직접 조작하지 않는다. 대신 영혼 + 실전 장비 + 스킬 + 펫 + 변신 조합으로 어떻게 싸울지를 선택한다.

  • 조합이 바뀌면 전투 방식이 바뀐다
  • 콘텐츠마다 유리한 세팅이 다르다
  • 성장 포인트를 어디에 쏟느냐가 전략의 핵심

전투 목표 3가지

  • 세팅의 재미 — 조합을 바꾸면 전투가 달라진다
  • 성장의 체감 — 숫자만 커지는 게 아니라 전투 방식 자체가 진화한다
  • 보는 재미 — 화면이 볼거리가 있어야 한다

목표 템포: 버섯커 키우기 ~ 세븐나이츠 키우기 사이

  • 숫자가 빠르게 튀어나오는 속도감
  • 스킬 연출이 화면을 과도하게 덮지 않는 균형
  • 방치 구간에서도 "뭔가 싸우고 있다"는 느낌 유지

  • 특정 빌드가 모든 콘텐츠에서 정답이 되지 않는다

8. 스탯 / 전투 공식 — 데미지 공식

전투력(CP) 계산 공식, 데미지 계산 흐름, 치명타 / 명중 / 회피 처리 규칙을 정리함.

전투력 (CP) 계산 공식

CP = ATK × 2.5
+ DEF × 1.5
+ HP × 0.001
+ CRIT × 30
+ ASPD × 20
+ BOSSDMG × 50
+ (AUTOREWARD, GOLDBONUS, EXPBONUS → 미반영)
⚠ 이 계수는 초안 방향값 — 정확한 가중치는 밸런스 단계에서 확정 후 PlayerGrowthConfigTable에 저장

데미지 계산 흐름

① BaseDMG = ATK × 스킬배율
② DefReduced = BaseDMG × (1 - DEF / (DEF + K))
   K = 방어 보정상수 (스테이지별 조정)
③ 치명타 발생 시:
   FinalDMG = DefReduced × (1 + CRITDMG)
④ 일반 공격 시:
   FinalDMG = DefReduced
⑤ 대상 보정 적용:
   보스 대상: FinalDMG × (1 + BOSSDMG)
   일반 몬스터: FinalDMG × (1 + MONSTERDMG)
보정상수 K는 StageBossTable / StageTable에서 스테이지별로 다르게 적용 가능

8. 스탯 / 전투 공식 — 밸런스

오프라인 보상 계산 공식 및 출시 후 밸런스 조정 포인트를 정리함.

오프라인 보상 계산

오프라인 시간 = MIN(실제 경과 시간, 12시간)
기본 보상 = 기본 사냥터 파밍률 × 오프라인 시간
